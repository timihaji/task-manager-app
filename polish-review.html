Making the task app feel like Things

Things doesn't actually do more than this app does — it just responds differently. Cards have weight. Buttons press down. Things settle instead of snap. Lists ripple instead of jump. This document is a feel-first review, with interactive demos so you can hold each change in your hands before we ship it.

·What "feel" actually means

Decompose the felt quality into ten concrete properties. Each translates to code; together they're the difference between "tool" and "object."

01
Settle, don't snap

Every position change overshoots ~1–3% and resolves, like a heavy object losing momentum.

02
Press-down on touch

Every interactive element compresses on press (~0.96) and releases. The body knows the click registered before the brain.

03
Optimistic, never waiting

State change is local-instant. Server sync is invisible.

04
Stagger when many move

Re-sorting cards step into position with ~25 ms delay between them, not all at once.

05
Magnetism on drag

Drop slots gently pull toward the cursor when within ~24 px.

06
Anticipation on hover

Adjacent cards subtly part before any action.

07
Cursor knows

grab on draggables, pointer on chips, text in inputs, not-allowed on locked.

08
Continuity over replacement

Modal scales from origin. Drawer slides. Card morphs. Never a hard cut where motion is possible.

09
Inertial scroll

Lists carry past a flick, gently rubber-band against the edge.

10
Haptics on touch

10 ms vibrate on long-press, completion, drop. Cheap. Transformative on phone.

Where we are now
The current app does almost none of this. CSS easing is in place (good), but no springs, no :active press, no stagger, no magnetism, only one cursor state (grabbing), modal slides from above (a notification idiom). The infrastructure for feel is missing, not the will.

·Top 5 changes (do these first)

Ranked by felt jump per hour invested. If time were finite, this is the order.

01 · ~30 min · CSS only

Press-down on every clickable

One CSS block adds :active { transform: scale(.97) } across cards, buttons, chips, checkboxes. The whole UI suddenly feels physical.

02 · ~1 hr · CSS only

Settle curves & cursor states

Define --ease-settle + --ease-out tokens and apply. Add cursor: grab. Things land softly instead of snap.

03 · ~1 hr · React + CSS

Reborn checkbox + optimistic strike

18 px circle, fill-and-tick animation. Strike happens synchronously with the click, not after the React render.

04 · ~2 hr · framer-motion

Stagger when lists re-sort

<motion.div layout> with spring + 25 ms delay per index. The list reads as fabric, not as React.

05 · ~2 hr · React + CSS

Magnetism + drop landing

Slots pull toward cursor when near. Dropped card scales 1.02 → 1.0. Drag becomes a conversation.

§1Feel & physics

The heart of the review. Each subsection has a felt deficit, the fix, the file/line where it lives, and — where it helps — an interactive demo so you can hold the change before we ship it.

17 items most CSS-only 3 items want framer-motion

1aAnimation curves are merely OK; nothing actually springsXS

Three different cubic-bezier curves in use across the app — none are spring-like. The card transform uses cubic-bezier(.2,.8,.2,1); the drawer slide uses cubic-bezier(.2,0,0,1); complete-fade uses cubic-bezier(.4,0,.2,1). None overshoot. Things' equivalent curves end with a tiny back-bounce that signals "this thing has mass."

Fix: define --ease-out (the drawer one — soft landing) and --ease-settle (a pre-computed elastic curve that overshoots ~5% and resolves). Apply --ease-settle to checkbox check, drawer entrance, modal entrance, drop preview, spawn-pulse end, list-item enter. Apply --ease-out to everything else.

Demo · Watch the difference between curves
stiff (current) · cubic-bezier(.4,0,.2,1)

Notice how only the bottom one overshoots and resolves — that's the settle. The brain reads it as mass.

1cOptimistic UI is structurally correct but not visibly optimisticS

Tasks update locally before Supabase confirms — so the data layer is already optimistic. But the animation doesn't honour it: clicking the checkbox waits for a re-render before the strike-through fires. Even a 16 ms delay reads as "did I click that right?"

Fix: Add a local done state in TaskCard that flips on the click event itself. The strike animation runs synchronously. Server sync surfaces as a tiny pending indicator, not a UI freeze.

1dDrag is good; landing is notS

The drag itself is well-tuned: opacity drops to 0.4, scales to .97, rotates 1.5 deg (styles.css:281). StackView clones with 3 deg tilt. Autoscroll has 60 px edge zones with proximity-scaled velocity. All Things-grade.

What's missing is the landing. When you drop, the card just appears in its new position. Things' drop scales briefly (1.02) at release, settles to 1.0 with the elastic curve, while neighbours that shifted to make room also settle, staggered.

1eMagnetic drop zonesS

Currently drop-zone activation is binary: you're either over a card (drop preview) or you aren't. Things has subtle attraction — when your cursor is near a slot, the slot widens to claim you.

Fix: in existing drag-over handlers (onCardDragOver in Column.jsx and StackView.jsx), compute distance from cursor to each candidate slot's centre. For slots within 24 px: apply a transform proportional to (1 - distance/24) × 4 px toward the cursor. The slot itself can grow slightly. ~30 lines.

Demo · Move your cursor across the zones
In proposed mode, the nearest slot lights up early and grows slightly. Drag becomes a conversation, not a binary state machine.

1fLists don't stagger when they re-sortM

When you complete a task in a column, the cards below slide up to fill the gap — but all at once with no offset. This is the single biggest "feels like React, not like an app" tell.

Fix: wrap rendered card lists in <motion.div layout> (framer-motion) with transition={'{type:'spring',stiffness:350,damping:35,mass:0.9}'} and a per-index delay so the ripple emanates from the action.

Demo · Click a checkbox to complete a task
In proposed mode, the cards below the completed one shift up with 25 ms delay each. Reads as a wave through fabric, not a frame-perfect React render.

1iModal & popover entrance directions are wrongXS

@keyframes su in styles.css:461: transform: translateY(-8px) — modal slides down from above. That's a notification idiom, not a modal. @keyframes popIn in styles.css:570 has the same problem for chip popovers.

Fix: swap to scale-from-origin. For chip popovers, transform-origin is the anchor; for modals, the click point. Both use --ease-settle so the entrance finishes with a 1% overshoot.

@keyframes modal-in {
  from { opacity: 0; transform: scale(.96); }
  to   { opacity: 1; transform: scale(1); }
}

@keyframes spawn-pulse at styles.css:309 does an outward shadow ripple in rgba(99,102,241,...) — purple, on a teal accent palette. Cards "ping" outward like a notification. Worse, the colour is wrong.

1lNo haptics on touchXS

The useTouchDrag hook in src/utils/useTouchDrag.js handles long-press-to-drag. Add a single line at the moment long-press succeeds: navigator.vibrate?.(10). Likewise on completion (10 ms) and on drop (15 ms). On phone, this is the difference between a website and an app.

§2Visual polish

Geometry that lets the physics breathe. Token-and-class-only changes; no JSX. ~30 min total.

2a–gThe token passS

One CSS block in styles.css changes nine tokens and unlocks the visual delta:

/* Default radius from 2px (sharp) to 6px (Things-soft) */
body[data-look="minimal"] { --rk: 6px; }   /* was 2px */

/* Card padding 7×8 → 10×12; title 12px → 15px */
.card { padding: 10px 12px; }
.card-title { font-size: var(--fs-md); line-height: 1.45; }

/* Chip text below the type scale → onto it */
.card-meta, .card-tag { font-size: var(--fs-xs); }   /* 9.5px → 11px */
.grp-name { font-size: 12px; font-weight: 700; }   /* was 10.5/600 */

/* List gap 3px (tense) → 6px (calm) */
.col-body { --card-gap: 6px; }

/* Dark-mode shadow softer, wider */
body[data-theme="dark"] {
  --shadow: 0 1px 2px rgba(0,0,0,.25),
            0 8px 24px rgba(0,0,0,.18);
}

/* Spawn-pulse colour from purple to accent */
@keyframes spawn-pulse {
  0% { box-shadow: 0 0 0 0 var(--accent); }
  100% { box-shadow: 0 0 0 8px var(--accent-dim); }
}

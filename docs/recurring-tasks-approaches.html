Cross-cutting features & UX (apply to whichever direction wins)

  • Edit-this vs edit-series modal on any rename/description change to a recurring task (Sunsama pattern).
  • Skip next occurrence as a one-click affordance — beats completing-to-clear-overdue.
  • Pause for vacation — pause all recurring between dates X & Y; auto-resume.
  • End conditions everywhere — until <date> and after N times, even on the simple chip.
  • Per-task rollover policy — auto-roll-to-today / leave-as-overdue / auto-skip after N days.
  • Snooze ≠ advance — snoozing one instance must not move the series next-fire.
  • Subtasks on spawn — explicit choice: copy fresh (unchecked) vs carry state.
  • ↻ glyph everywhere — .card, .scard, .list-item all show the recurrence indicator (consistency rule).
  • Completion history on every card — "7× in last 30 days" tooltip even without Habit Mode.
  • NL parser as a layer, not a mode — even if we pick the chip UI, parsing in QuickEntry is cheap power.
  • Pre-generate one instance ahead (Sunsama-style) — tomorrow's recurring tasks already in the Timeline tonight.
  • Auto-tag "routine" so Stack / List filters work without extra effort.
  • "Roughly at" time hint (Sunsama) — orders the day without strict timeboxing.
  • Time-zone safety — current code treats dates as local strings; recurrence math should pin to local midnight explicitly.
